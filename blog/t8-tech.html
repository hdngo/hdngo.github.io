An Introduction to the Internet

Something that I know I'll need to remember in the midst of learning how to program and develop websites is how the Internet works. I mean let's be honest, can you really give me a flawless explanation as to how everything works if I asked you to right now? If you can, good for you, if you can't, no worries, becauseI'm going to just give cover the basics as basic as I can. And don't worry, I'll be working on this particular blog incrementally so that I can simplify it as much as possible.

The Internet is like a wire.

  • Servers are connected to the Internet; the Internet is a super computer
  • Web pages are on a server's hard drive
  • Every server has an Internet protocol address
  • Because IP address names are hard to remember, we give them names
  • Our own computers are not servers, b/c they’re not directly connected to the Internet
  • Our computers are clients, as they’re connected by an Internet service provider (ISP)
  • Whenever an email, picture, etc. is sent across the Internet, it's broken into packets which are later put back together

IP Addresses and Routers

  • Everything corrected to the Internet has an IP address, even phones
  • routers direct your packets around the Internet
  • Packets are wrapped in "layers"
  • The 1st layer is the IP address, it's sent to the next router and adds that IP address; each router adds another layer/IP address
  • when the server sends back info, it sends back packets with identical wrapping, each router unwraps the layer

The Internet vs The Web

  • The Internet is a system of interconnected computer networks using the same rules (protocols) to talk to each other
  • It was initially used for exchanging, emails and remote logins
  • It's the ‘network of networks'
  • The Web is an application built on top of the Internet
  • It's an information sharing app that allows you to read and interact with web pages through a web browser

Web Browser

  • Desktop software applications allowing you to browse the Internet to look at a website without a browser
  • Web browsers take code and represent it
  • World Wide Web Consoritum (W3C) sets forth browser guidelines but can’t enforce them
  • One big problem is backwards browser compliance

Web Servers

  • A computer that can be accessed through the Internet
  • Web servers receive your requests for web pages and put together the parts, then send it back
  • Windows comes with Internet Information Services (IIS) and Apple computers come with Apache - web server softwares

Websites vs Web Applications

  • Websites are static in that they're not updated all that often
  • Web applications are dynamic and ever changing
  • They rely on you to interact with them
  • They're built with HTML, CSS, etc, but also use programming languages like PHP, Ruby, Python and frameworks like Rails, Django and CakePHP
  • They almost always use databases - therefore they’re dynamic

Domains

  • A domain name is a shortcut to a computer hooked up to the Internet
  • Top level domains are .com, .net, .org, etc.
  • Second level domains: site name ie. skill crush.com
  • Domain names are the nicknames for IP addresses
  • ICANN - the Internet Corp for Assigned Names and Numbers controls the ins and outs of naming on the Internet, and are responsible for licensing everyone who sells domains

DNS

  • The phone book of the Internet
  • Domain Name System
  • DNS servers are computers that keep record of domain names and IP addresses
  • When you make a web request, your computer asks the DNS server for the IP address numbers get into the DNS phone book when you register a domain name
  • When you change a domain name or web server, DNS can take time to update - propagation delay

IP Address

  • Your computer’s address on the Internet
  • Each number in your IP address: X.X.X.X can go from 0 to 255
  • IPv6 IP addresses are super long

Introduction to DNS and Creating a New Website

  • Buy the domain name
  • Registrars get licenses to sell domain names
  • Find a host for the website - if you buy domain and hosting service from the same company, you don’t need to worry about DNS
  • Specify name servers for your domain
  • Point the domain name at the IP address (using the registrar, host or another) point domain.com to your new IP address
  • www is the host name, IP given by your host
  • Using * as a hostname -> everything.domain.com points to the IP address
  • @ as the hostname —> domain.com without the www part
